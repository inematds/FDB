Conceitos fundamentais

  • Modelo relacional: tabelas, linhas, colunas, chaves.
  • Integridade: PK, FK, UNIQUE, NOT NULL, CHECK.
  • Normalização: reduzir redundância (1FN → 3FN) e dependências.

SQL essencial

  1. Crie a tabela cliente com PK, nome e email único.
  2. Insira 3 clientes e consulte apenas os que têm email.
  3. Crie a tabela pedido com FK para cliente.
CREATE TABLE cliente (
  id SERIAL PRIMARY KEY,
  nome TEXT NOT NULL,
  email TEXT UNIQUE
);

CREATE TABLE pedido (
  id SERIAL PRIMARY KEY,
  cliente_id INT REFERENCES cliente(id),
  total NUMERIC(10,2) NOT NULL
);
